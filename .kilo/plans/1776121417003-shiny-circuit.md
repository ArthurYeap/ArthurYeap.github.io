# Plan: Fix Games Grid Layout

## Problem

The "games-left" section grid isn't working properly. After analyzing the code, I've identified several issues:

### Issues Found

1. **Second game (Cyberpunk 2077) missing Bootstrap grid**: 
   - First game (Uncharted 4) uses proper Bootstrap grid: `.row` > `.col-5` / `.col-7`
   - Second game (Cyberpunk 2077) at lines 194-227 uses raw `<div class="games">` without any grid structure

2. **CSS conflict in `.name` class** (interest.css:181-182):
   ```css
   align-items: flex-end;
   align-items: center;  /* This overwrites the previous line */
   ```

3. **Fixed height causing issues** (interest.css:200-201):
   - `.games-left` has `height: 80vh` which forces a fixed height

## Solution

### 1. Add Bootstrap grid to Cyberpunk 2077 section

Wrap the second game content in Bootstrap row/column structure to match the first game:

```html
<section id="games" style="margin-top: 5rem">
    <div class="container-fluid">  <!-- Already exists -->
        <div class="row">
            <div class="col-12 col-md-5">
                <div class="img">
                    <img ... />
                </div>
            </div>
            <div class="col-12 col-md-7">
                <div class="games-left">
                    <div class="name">
                        ...content...
                    </div>
                </div>
            </div>
        </div>
    </div>
</section>
```

### 2. Fix CSS conflict in `.name` class

Remove conflicting `align-items` properties. Keep only one:

```css
.name {
    display: flex;
    flex-direction: column;
    width: 70%;
    align-items: center;  /* Keep this, remove flex-end */
    justify-content: center;
    scale: 1.3;
}
```

### 3. Remove/fix fixed height on `.games-left`

Either remove `height: 80vh` or change to `min-height` to allow content to flow properly.
